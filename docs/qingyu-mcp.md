# 轻语 MCP

轻语（QingYu）桌面版内置一个默认关闭的 MCP 服务。MCP 客户端调用的是轻语提供的文档、设置和同步工具；客户端不会获得本机文件系统 API，也不能提交任意绝对路径。

## 工作方式

```text
MCP 客户端 ──stdio──> qingyu-mcp ──本机私有 IPC──> 轻语应用服务 ──> 当前笔记目录
```

- `qingyu-mcp` 是随桌面版打包的 stdio 桥接器。它只转发 MCP 协议消息，不实现文档读写，也不保存业务数据。
- macOS 和 Linux 使用仅限本机的 Unix domain socket；Windows 使用本机 named pipe。不监听 HTTP 或 TCP 端口。
- MCP 只接受轻语签发的 `workspaceId`、`folderId` 和 `documentId` 等不透明标识。所有文档操作最终由轻语应用服务完成。
- MCP 文档工具的唯一文件访问边界是轻语当前选择的笔记目录。独立打开或聚焦单个 Markdown 文件不会改变该边界。
- 当前笔记目录缺失或不可用时，文档工具会失败关闭；应用设置与同步策略仍可按各自权限使用。

本机 MCP 服务只在桌面版中提供。移动端可以查看和编辑应用级 MCP 策略，但不会注册本机 MCP 服务、IPC、工具或文件系统能力；Web 编辑器不提供这些能力。

## 应用配置和权限

轻语只维护一份应用级 MCP 策略，并把它保存在可携带应用设置 `settings.json` 的 `mcp` 字段中。策略包含：

- 是否启用 MCP；
- 文档、应用设置和同步工具的读写权限；
- 确认、试运行、删除和同步执行策略；
- 请求、响应、文档、速率、并发和审计限制。

应用设置不保存目录白名单、绝对路径、连接令牌、签名密钥、socket 路径或审计记录。IPC、审计和其他仅运行时状态保存在本机应用数据的 `mcp-runtime/` 目录中，不参与设置同步。

同一份应用策略适用于所有 MCP 客户端，不按客户端或笔记目录分配单独权限。通过同步或其他设置入口更新该策略后，桌面 MCP 运行时会重新加载；默认策略保持关闭。

## 启用

1. 如需使用文档工具，在轻语设置中选择当前笔记目录。
2. 打开“设置 > MCP”。
3. 配置所需权限和操作策略。
4. 启用应用 MCP 策略。
5. 在启用后出现的“客户端连接”区域选择配置格式并复制。

设置页显示当前笔记目录和本机 IPC 运行状态，不提供端口、令牌或额外目录授权入口。没有当前笔记目录时仍可编辑应用策略，但文档工具不可用。移动端只显示策略设置，不显示客户端连接配置。

## 客户端配置

轻语桌面版会读取当前安装包内 `qingyu-mcp` 的真实路径，并在 MCP 启用后生成可直接复制的配置。默认提供 Codex TOML：

```toml
[mcp_servers.qingyu]
command = "/Applications/QingYu.app/Contents/MacOS/qingyu-mcp"
```

也可以切换为兼容 `mcpServers` 的通用 JSON：

```json
{
  "mcpServers": {
    "qingyu": {
      "command": "/Applications/QingYu.app/Contents/MacOS/qingyu-mcp"
    }
  }
}
```

示例中的 macOS 路径仅用于说明；设置页生成的是当前系统和当前轻语安装位置对应的路径，无需手动查找或替换。“复制给 AI 工具”会同时复制一段安装说明和所选格式的配置，但轻语不会直接修改其他应用的配置文件。

客户端直接执行配置中的绝对路径，因此不需要把 `qingyu-mcp` 或可选的 `markra` 命令安装到 `PATH`。复制后的配置可以一直保留：关闭 MCP 不会删除或改写 Codex、Claude 等第三方客户端配置。

配置中不要添加笔记目录、`QINGYU_MCP_TOKEN`、`QINGYU_MCP_URL` 或其他 URL、端口和密钥。桥接器会先连接已经运行的轻语服务；只有应用设置中的 MCP 已启用且服务尚未运行时，才会直接启动安装包内的轻语主程序并进入无窗口 `mcp serve` 模式。该模式不会创建编辑器窗口、激活 Dock 图标或抢占当前焦点，并在最多 15 秒内进行有限重连。用户之后主动打开轻语时，现有进程才会创建或恢复普通编辑器窗口。

如果 MCP 未启用或从未启用，桥接器会以非零状态退出并返回 `mcp_disabled`，提示在轻语设置中启用 MCP，同时不会启动应用。如果本机 `settings.json` 无法安全读取或 MCP 配置结构损坏，会返回 `mcp_config_unavailable`；如果随包主程序缺失或不是普通文件，会返回 `app_launch_failed`。这些错误都不会回退到 PATH、`open -a`、HTTP 或任意客户端命令。连接中断后不会自动重放工具调用，因为写操作的结果此时可能无法确定。

## 可用能力

工具按应用级权限动态提供：

- 工作区：返回当前笔记目录的安全标识和根文件夹标识；未选择有效笔记目录时不返回文档工作区。
- 文档：列出、搜索、读取、新建、更新、移动、重命名和删除 Markdown 文档。
- 应用设置：读取或原子更新轻语明确公开的外观、语言、编辑器、忽略规则和 PDF 导出设置字段。
- 同步：读取脱敏配置和状态，更新 WebDAV/S3 非敏感配置，写入或清除凭据，测试连接以及启动同步。

MCP 不提供 `workspace_open`、`workspace_close`、目录选择或任意路径工具，也不公开 MCP resources 或 prompts。设置字段采用白名单，未公开的内部设置不能通过 MCP 修改。

## 文件边界和撤权

- 新建操作要求 `workspaceId + parentFolderId + name`，其他文档操作使用轻语签发的不透明 ID。客户端不能使用绝对路径绕过边界。
- 轻语只保留当前笔记目录的受限目录句柄，并在每次操作时重新校验目录身份、相对路径和符号链接边界。
- 切换、移除或失去当前笔记目录会更新目录能力并使旧工作区、文档及预览标识失效；独立文件窗口的焦点变化不会产生这一效果。
- 文档 ID、游标和预览令牌使用仅存在于当前轻语进程内的随机签名材料。重启轻语后旧标识会失效，客户端重新调用 `workspace_list` 和文档列表即可获得新标识；MCP 客户端配置不需要改变。
- 符号链接、reparse point、路径穿越、绝对路径、受保护目录、未知请求字段和隐式覆盖都会被拒绝。
- 关闭应用 MCP 策略会停止本机 IPC 服务并取消活动会话及未使用的预览。当前笔记目录不可用只会撤销文档能力，不会改写应用策略。

## 写入安全

- 文档和设置读取结果带有修订号。更新必须提交预期修订号；内容变化时会返回修订冲突，不覆盖较新的数据。
- 确认策略可设为从不确认、仅破坏性操作确认或所有写入确认。确认界面由轻语展示。
- 试运行可应用于高风险操作或所有写入。需要试运行的请求先返回预览和短期预览令牌，再用该令牌提交。
- 删除策略可配置为系统废纸篓、轻语回收站或永久删除。默认使用系统废纸篓。
- 选择轻语回收站后，可将自动清理设为永不、7 天后、30 天后或 90 天后；默认 30 天。该设置只清理轻语私有回收站，不影响系统废纸篓，也不限制用户以后手动清空回收站。
- 审计存储不可用时，写操作会在实际变更前失败关闭；只读操作仍可继续。

## 敏感信息

MCP 传输不使用钥匙串、Bearer Token 或持久签名密钥。Unix socket 文件权限限制为当前本机用户，Windows named pipe 使用当前进程的本机访问控制。

同步凭据仍是只写输入：读取同步配置只返回是否已配置凭据，不会回显用户名、密码、访问密钥或秘密密钥。审计记录保留工具名、结果、时间和安全逻辑标识，但不会记录同步秘密、文档正文、进程签名材料或项目绝对路径。
