# QingYu MCP 默认传输限制简化设计

## 背景

轻语的 MCP 设置页目前直接展示文档、请求、响应大小，以及限流、并发和超时等七项数值。它们属于 MCP 服务的运行保护边界，默认值已经适合正常使用；直接暴露会让普通用户误以为启用 MCP 前必须理解和调整这些参数。

## 方案比较

1. **从设置页删除传输限制控件（采用）**：界面最简洁，后端继续使用并校验默认限制，运行保护不变。
2. **折叠为高级设置**：保留调节能力，但仍增加一个普通用户不需要理解的入口，也保留误配置风险。
3. **从配置模型和后端完全删除限制字段**：会取消可持久化的运行边界并扩大跨层改动，不符合本次仅简化设置页的目标。

## 设计

- 从桌面和紧凑版 MCP 设置页删除“传输限制”区块及七个数字输入框。
- 删除仅供该区块使用的界面字段映射和多语言文案。
- 保留 `McpConfig` 中的七个限制字段、`defaultMcpConfig()` 默认值、归一化范围和 Rust 运行时限制逻辑。
- 不增加“高级设置”入口，不改变 MCP 权限、操作策略、审计、客户端快捷配置或同步行为。
- 已持久化配置仍按现有规则读取和归一化；本次不引入迁移。

## 测试

- 新增界面测试，确认默认配置仍包含七项保护值，但 MCP 设置页不再渲染对应数字输入框。
- 更新现有设置页覆盖清单，避免继续要求这些控件可见。
- 运行 MCP 设置定向测试、应用包测试、类型检查和生产构建。

## 验收标准

- 用户在 MCP 设置页看不到“传输限制”标题和七项数值设置。
- 新建或归一化后的 MCP 配置继续使用现有默认限制。
- MCP 运行时的大小、速率、并发和超时保护保持有效。
