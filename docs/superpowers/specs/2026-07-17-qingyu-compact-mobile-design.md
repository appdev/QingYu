# QingYu Compact 移动端设计

## 状态

本设计已于 2026-07-17 通过交互确认。它描述 QingYu 在手机尺寸和 Tauri 移动运行时中的首版产品边界、界面结构、工作区模型、自动保存与项目级同步行为。

## 目标

在当前 QingYu 应用中增加适合手机使用的 Compact 产品模式，而不是完整移植桌面端全部能力。Compact 模式共享现有 Markdown 编辑、文件操作、项目配置和远程同步实现，只替换移动端的信息架构与交互外壳。

首版必须满足：

- 手机上以 WYSIWYG 编辑为核心；
- 使用一个由 App 管理的固定内部工作区；
- 用户可以完全本地使用，也可以配置 WebDAV 或 S3 同步；
- 目录和设置都是覆盖编辑器的全屏页面；
- 同步引擎、删除传播、checkpoint 和冲突副本行为不产生移动端分叉；
- 当前系统可以通过缩小桌面窗口或浏览器视口验证 Compact UI；
- 桌面宽度下保持现有产品行为。

## 非目标

首版不包含：

- AI、AI 提供商或 Web 搜索；
- Markdown 源码模式和编辑/预览分屏；
- 多工作区、最近目录、目录选择器或工作区切换；
- App 内清空工作区、重置工作区或删除云端数据；
- 移动端专属同步引擎、冲突解决器或同步协议；
- 标题驱动的文件自动命名或标题变化后的自动重命名；
- 模板、导出、键盘快捷键和桌面网络代理等完整桌面设置；
- Web Runtime 中的真实 WebDAV/S3 项目同步；
- 同步空目录或符号链接。

## 当前实现基线

设计以当前代码为准：

- 同步配置属于项目，唯一事实源是 `<project>/.qingyu/config.json`；
- manifest 和非秘密状态位于 `<project>/.qingyu/sync/`；
- `.qingyu/` 和旧的 `.markra-sync/` 都是受保护控制目录，不进入文件树、搜索、监听、备份或双向同步；
- WebDAV 与 S3 共享现有远程同步引擎；
- 项目配置字段原子写入，编辑同步设置时自动触发暂停，退出设置后应用最终配置；
- 当前桌面应用已有分钟级自动保存；它保存已有路径的脏文件，但不发出项目同步的 `save` 触发；
- 文件树创建先输入名称，空名称取消；快速草稿和未命名文档仍使用现有 `Untitled.md` 语义；
- Web Runtime 明确不支持真实项目远程同步。

## 产品模型

### 同一个应用，两个界面外壳

共享控制层负责文档、文件、编辑器、设置、项目配置和同步状态。Desktop Shell 与 Compact Shell 只负责不同的界面编排：

```text
文档 / 文件 / Milkdown / 设置 / 项目配置 / 同步协调器
                         │
              ┌──────────┴──────────┐
              │                     │
         Desktop Shell        Compact Shell
          现有桌面布局          手机信息架构
```

Compact 不是只给桌面 CSS 增加换行，也不是新的独立前端应用。它应由独立组件构成，但依赖共享的动作与状态接口。

### Compact 判定

满足任一条件即进入 Compact：

- Tauri 运行时明确标记为移动端；
- 当前视口宽度不大于 720 CSS 像素。

移动运行时在横屏时仍强制 Compact。桌面 Tauri 缩窗进入 Compact 后继续使用当前打开的桌面项目；Web 浏览器缩窗使用 Web 文件适配器，只验证 Compact UI 和本地 Web 能力。

判定不依赖 User-Agent。运行时能力与视口条件应被封装，组件不能各自重复判断平台。

## 架构边界

### 共享应用控制层

从当前大型 `App.tsx` 中提取 Compact 和 Desktop 都需要的状态与动作接口，包括：

- 当前文档、打开文档和上次文档；
- 文件树数据与文件/文件夹操作；
- WYSIWYG 编辑器状态；
- 本地保存、显式保存和保存状态；
- 项目根目录与项目配置；
- 同步状态、立即同步和设置编辑会话；
- 应用语言、主题和可用能力。

提取不得顺带重写已有业务逻辑。Desktop Shell 应先在新接口上保持现有输出，再接入 Compact Shell。

### Compact 组件边界

Compact 至少拆分为以下职责单一的组件或模块：

- `CompactAppShell`：页面栈、系统返回和安全区域；
- `CompactEditorScreen`：顶栏、欢迎状态、编辑器和键盘工具栏；
- `CompactFileBrowserScreen`：全屏目录、搜索与文件操作入口；
- `CompactMoveTargetScreen`：文件或文件夹的移动目标选择；
- `CompactSettingsHome`：分组设置列表；
- `CompactSettingsDetail`：单个设置分类；
- `CompactSyncStatusScreen`：本地、已配置和错误状态；
- `CompactSyncFormScreen`：首次配置与修改配置；
- `useCompactAutoSave`：移动端防抖、生命周期补存和状态反馈；
- 移动工作区运行时适配器：只负责解析 App 私有持久目录。

命名可在实现计划中按现有模块习惯调整，但上述职责不能重新集中回 `App.tsx`。

### 包边界

- Compact React 组件和共享控制逻辑放在 `packages/app`；
- Milkdown 行为继续放在 `packages/editor`；
- Markdown 路径和资产逻辑继续放在 `packages/markdown`；
- 跨层小型类型和纯工具放在 `packages/shared`；
- Tauri 文件系统和移动运行时桥接留在 `apps/desktop/src/lib/tauri` 与对应 Rust 模块；
- 不为此功能引入新的 UI 框架或状态管理框架。

## 工作区模型

### 真机工作区

移动端首次启动时，在 App 私有的持久数据区域创建唯一工作区。产品文案可称为“App 内部目录”，实际位置不能使用可能被系统回收的 Cache 目录。

```text
App 私有持久目录/
├── 用户的 Markdown、图片和附件
└── .qingyu/
    ├── config.json
    └── sync/
```

工作区创建后不再显示目录选择器。用户可以在其中新建完整目录层级。App 不保存多个移动工作区记录，也不提供切换入口。

### 本地模式

没有 `.qingyu/config.json` 时，固定工作区就是本地模式：

- 所有编辑与文件操作可用；
- 不发生同步网络请求；
- 设置显示“本地使用”和“配置同步”；
- 用户可以永久保持该状态。

### 清除 App 数据

清除 App 数据是操作系统层面的用户行为，不是 QingYu 功能。清除后本地工作区和配置会消失，下次启动重新创建空工作区。QingYu 不因此连接、修改或删除云端内容。

## 启动与文档恢复

移动端记录上次打开文档在固定工作区内的相对路径。启动流程为：

1. 创建或解析固定工作区；
2. 加载文件树和项目配置；
3. 验证上次文档仍属于工作区且存在；
4. 存在则打开该文档；
5. 不存在、已删除或从未打开过文档时，显示欢迎空状态。

欢迎状态提供两个主要动作：

- “新建文档”：进入现有名称输入流程，创建后打开编辑器；
- “配置同步”：进入同步状态或配置页。

上次文档失效不是启动错误，不显示故障弹窗。

## Compact 导航

Compact 使用全屏页面栈：

```text
编辑器
  ├─ 全屏目录
  │    └─ 全屏移动目标选择
  └─ 设置首页
       └─ 设置详情或同步页面
```

- 编辑器是根页面；
- 目录从左侧入口进入并完全覆盖编辑器；
- 选择文件后返回编辑器；
- 设置从编辑器“更多”进入；
- 设置详情返回设置首页，设置首页和目录返回编辑器；
- Android 系统返回键遵循页面栈；
- iOS 支持边缘返回；
- 编辑器根页面再次返回时交给系统，不增加自定义退出流程；
- 不使用永久底部主导航。

在全屏页面切换期间，共享文档状态保持存活，不能因为 Compact 页面卸载而丢失编辑内容。

## 编辑器

### 顶栏

顶栏采用极简结构：

- 左侧：全屏目录入口；
- 中间：当前文件名和本地保存状态；
- 右侧：“更多”。

“更多”首版包含：

- 保存；
- 文档内查找；
- 文档历史；
- 立即同步，未配置时显示“配置同步”；
- 设置。

不出现 AI、源码、分屏和桌面窗口操作。

### 编辑模式

Compact 只挂载 Milkdown WYSIWYG。源码和分屏不是隐藏按钮后仍运行的功能，而是在 Compact 能力清单中不可用。

### 键盘格式工具栏

软键盘出现时，在键盘上方显示单行、可横向滚动的格式工具栏：

- 撤销、重做；
- 正文与标题级别；
- 粗体、斜体、删除线、行内代码；
- 链接；
- 无序列表、有序列表、任务列表；
- 引用、代码块；
- 插入图片；
- 收起键盘。

复制、粘贴和文本选择继续使用系统菜单，不增加一套与原生选择手柄冲突的浮动工具栏。

## 文件与目录

### 全屏目录

目录页顶栏提供返回、工作区状态、新建和搜索。正文复用现有文件树数据、过滤和排序规则，按完整层级显示文件夹。

- 点击文件夹展开或收起；
- 点击文件前先执行当前文档补存，成功后打开目标文件并返回编辑器；
- 补存失败时不切换文档，返回编辑器错误状态并允许重试；
- 行尾“更多”和长按打开同一个底部操作面板；
- 不依赖悬停或右键；
- 不使用触屏拖拽完成移动。

### 支持的操作

文件支持：

- 重命名；
- 移动；
- 删除。

文件夹支持：

- 新建文件；
- 新建子目录；
- 重命名；
- 移动；
- 删除。

移动操作进入全屏目标目录选择器。删除使用明确的手机确认面板，底层删除和同步传播语义保持不变。

### 命名

Compact 新建文件使用当前文件树的名称输入流程：

- 用户先输入名称；
- 空名称取消创建；
- 创建成功后进入编辑器；
- 保留当前扩展名处理、名称冲突和错误提示；
- 不从正文标题生成文件名；
- 后续标题变化不自动重命名物理文件。

桌面端快速草稿和其他现有入口不因 Compact 实现而改变。

## 移动自动保存

### 行为

Compact 对已有路径的当前文件启用快速本地自动保存：

- 内容变化后标记为未保存；
- 停止输入 1.5 秒后写入；
- 切换文档前立即补存；
- 进入目录或设置前立即补存，但打开这些页面本身不丢弃当前编辑状态；
- App 进入后台前立即补存并持久化现有工作区草稿状态；
- 同一文档的并发保存必须合并，旧结果不能覆盖更新内容。

Compact 创建流程先创建有名称的文件，因此正常新文档具有保存路径。现有桌面未命名草稿恢复机制继续保留，不为 Compact 改写全局文档模型。

### 与同步的关系

快速自动保存复用当前本地自动保存通道，不调用 `notifyDocumentSaved`，因此不产生项目同步的 `save` 触发。

用户从“更多”执行显式保存时，继续走现有保存包装器；当项目配置启用 `autoSyncOnSave` 时，该显式保存可以产生现有的保存后同步。

项目打开、退出同步设置、立即同步和定时同步的触发保持不变。

### 状态与失败

文件名附近以文字和图标显示：

- 未保存；
- 保存中；
- 已保存；
- 保存失败。

状态不能只依赖颜色。保存失败使用持续可见的错误条和一个“重试”动作，不能只显示短暂 Toast。失败时保留编辑状态和脏标记，不触发同步。

桌面端继续使用现有分钟级自动保存偏好，不启用 1.5 秒 Compact 防抖。

## 项目级同步

### 数据源

移动端不创建应用级同步设置。固定工作区就是当前项目根目录，Compact 同步 UI 直接使用现有项目配置运行时、配置 revision、编辑会话、状态和同步协调器。

凭据继续按当前方案明文写入 `.qingyu/config.json`。Compact UI 显示简短安全说明；控制目录不会上传到远端。

### 页面状态

#### 本地模式

当配置缺失时显示本地模式说明和一个“配置同步”按钮，不主动创建配置或连接网络。

#### 首次配置

点击“配置同步”后创建当前项目的默认配置并进入单列配置表单。表单包含：

- 启用状态；
- WebDAV 或 S3；
- 远端目录；
- 当前提供商字段；
- 保存后同步；
- 同步间隔；
- 测试连接。

字段继续使用现有原子 patch 和 expected revision。页面的“完成”只结束编辑会话并返回，不新增另一套保存协议。离开后由现有 `settings-exit` 流程加载并应用最终配置。

#### 已配置

状态页优先显示：

- 是否启用；
- 提供商；
- 最近尝试与最近成功时间；
- 运行、成功或失败状态；
- 传输和冲突摘要；
- “立即同步”；
- “修改配置”。

配置字段位于独立表单页，不与状态摘要堆叠在同一长页面。

#### 配置错误

配置不完整时保留可识别字段，进入普通配置表单补齐。

配置损坏或版本不支持时：

- 页面显示明确错误类型和原因；
- 最多提供一个“配置同步”按钮；
- 点击进入空白或默认配置表单，由用户重新填写；
- 该表单先维护本地恢复草稿，不对无法解析或不受支持的原配置执行字段 patch；
- 成功完成时，底层先保留原配置备份，再安全替换为当前版本配置；
- UI 不显示重置、打开原始配置、导出配置或 JSON 编辑器。

该恢复只替换项目同步配置，不删除笔记、其他工作区文件、manifest 或云端对象，也不属于工作区重置。

### 设置编辑会话

Compact 必须保持当前同步编辑语义：

- 进入配置表单后，保存后同步和定时同步暂停；
- 对正常加载或不完整的配置，每个可识别字段变化立即规范化并原子持久化；
- 对损坏或版本不支持的配置，仅在用户完成恢复表单时备份并原子写入完整的新配置；
- 填完最后一个字段不会在用户仍编辑时自动连接；
- “完成”、返回或关闭设置结束会话；
- 未修改时不运行同步；
- 修改后仅在最终配置有效且启用时应用并运行一次；
- “立即同步”仍是显式例外，只使用当前已持久化且有效的 revision。

### 引擎不变

Compact 不修改：

- WebDAV/S3 后端；
- 双向扫描与 manifest；
- 上传、下载和删除传播；
- 原子下载和 checkpoint；
- 首次同步基线；
- 冲突副本命名与处理；
- 同项目触发合并和跨项目串行执行；
- 失败中的安全错误与秘密信息过滤。

移动端只展示同步结果，不增加专属冲突向导。

## Compact 设置范围

Compact 使用能力清单筛选设置，不直接把桌面的 16 个分类压进手机页面。

| 桌面分类 | Compact 首版 |
| --- | --- |
| 常规 | 保留语言等移动端有效项 |
| 存储 | 保留内部工作区与占用信息，只读 |
| 同步 | 保留并使用移动页面结构 |
| 外观 | 保留系统、浅色、深色和移动端有效显示项 |
| 编辑器 | 保留 WYSIWYG 字体、字号等有效项 |
| 拼写检查 | 运行时支持时保留 |
| AI、提供商、Web 搜索 | 隐藏 |
| 视图 | 隐藏，Compact 固定 WYSIWYG |
| 备份、模板、快捷键、导出、网络 | 首版隐藏 |
| 运行日志 | 不作为普通分类；首版不增加诊断页面 |

设置首页分为：

- 工作区：同步、存储；
- 编辑体验：外观、编辑器、拼写检查；
- 应用：常规、关于。

App 内不出现工作区重置、工作区切换、清空本地内容或删除云端内容的设置。

## 错误处理

- 固定工作区初始化失败：显示阻断故障页、错误原因和“重试”，不提供清空数据；
- 上次文档不存在：欢迎空状态；
- 创建、重命名、移动或删除失败：保持原文件树状态并显示可操作错误；
- 自动保存失败：保留脏内容和持久错误条；
- 离线：本地编辑和保存继续，显示“等待网络”，不重复弹窗；
- 同步失败：不回滚成功的本地保存，状态页显示安全摘要并允许立即同步重试；
- 连接测试失败：保留表单输入，在表单中显示错误；
- 冲突：展示现有引擎结果，不改变冲突文件。

## 移动端基础规则

- 主要触控区域至少为 44×44 CSS 像素；
- 适配状态栏、底部手势区域和 `safe-area-inset-*`；
- 软键盘出现时保持光标与格式工具栏可见；
- 使用真实可滚动区域，不靠缩小整个页面容纳键盘；
- 不依赖 hover、右键或精确拖拽；
- 支持系统字体缩放、明暗主题和减少动态效果；
- 页面转场使用短距离滑动，减少动态效果时取消；
- App 私有工作区不申请外部存储权限；
- 运行、保存、错误和选中状态不能只靠颜色表达。

## 实施顺序

1. 提取共享应用控制层，让 Desktop Shell 在新边界上保持现有行为；
2. 建立 Compact Shell、页面栈、编辑器顶栏和全屏目录；
3. 接入移动固定工作区、启动恢复与欢迎状态；
4. 接入 1.5 秒本地自动保存和移动生命周期补存；
5. 接入 Compact 设置清单、同步状态页和同步表单；
6. 审核并隔离桌面菜单、窗口、更新、Shell 和文件选择等移动端不支持能力；
7. 完成 Android/iOS 运行时验证和同步验收。

每一阶段都必须保持 Desktop Shell 可构建和可测试。不得先复制整个 `App.tsx` 再分别演化。

## 测试策略

### 单元测试

- Compact 判定：移动运行时、720px 边界和桌面宽度；
- 页面栈与 Android 返回；
- 设置能力清单；
- 相对路径的上次文档恢复和失效回退；
- 自动保存防抖、并发合并、补存、失败和重试；
- 自动保存不调用 `notifyDocumentSaved`；
- 显式保存保持现有保存后同步；
- 同步状态到 Compact 页面状态的映射；
- 配置缺失、不完整、损坏和版本不支持；
- 错误状态最多只有一个“配置同步”动作；
- 文件创建继续要求名称且不使用正文标题。

### 组件和视口测试

覆盖至少：

- 390×844；
- 430×932；
- 手机横屏；
- 软键盘打开与关闭；
- 长文件名、深层目录、长设置表单；
- 深色主题和系统字体放大；
- 安全区域与减少动态效果。

验证目录和设置是真正的全屏页面，并且没有横向布局溢出。

### 桌面回归

- 桌面宽度不进入 Compact；
- Desktop Shell 的文件、编辑器、设置、显式保存和分钟级自动保存不变；
- 快速草稿和 `Untitled.md` 语义不变；
- 当前多项目和单文件编辑行为不受移动固定工作区影响。

### 项目同步验证

- 运行项目配置、同步协调、WebDAV/S3 和 Rust 引擎测试；
- 验证编辑设置时自动触发暂停，完成后只应用一次；
- 验证 Compact 本地自动保存不产生 `save` 同步；
- 验证手动、项目打开、设置退出和定时触发；
- 验证配置错误的单按钮恢复；
- 验证 `.qingyu` 和 `.markra-sync` 保持双向排除；
- 条件允许时运行 `pnpm test:s3-sync:live`；
- WebDAV 在可用真实测试服务上执行现有验收。

### 命令门禁

实现完成前至少运行：

```text
cargo test --manifest-path apps/desktop/src-tauri/Cargo.toml
pnpm test
pnpm typecheck:test
pnpm build
```

移动运行时变更还需要适当的 Tauri Android/iOS 构建或真机 smoke test。

## 验收标准

- 手机尺寸不存在横向布局溢出；
- 编辑器、目录和设置为互斥的全屏页面；
- 目录选择文件后回到编辑器；
- 启动恢复上次有效文档，否则显示欢迎状态；
- 移动端只有 WYSIWYG，没有 AI、源码和分屏入口；
- 新文件先命名后创建，不从正文标题自动命名；
- 1.5 秒快速保存可靠，失败不会静默丢失内容；
- 快速保存不触发网络同步；
- 固定工作区无需配置即可本地使用；
- 项目级 WebDAV/S3 配置、状态和现有同步引擎被复用；
- 配置错误只显示原因和一个进入同步选项的按钮；
- 冲突、删除传播和 checkpoint 与桌面端一致；
- App 内没有工作区重置、切换、清空或云端删除入口；
- 桌面宽度和 Desktop Shell 行为没有回归。

本设计没有待定产品决策。实现中的组件命名和文件拆分可以在实施计划中细化，但不得改变上述产品行为、运行时边界或同步语义。
