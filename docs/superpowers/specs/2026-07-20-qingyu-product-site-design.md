# 轻语产品官网设计

## 状态

本设计已于 2026-07-20 通过交互确认。它定义轻语单页产品官网的定位、内容结构、视觉语言、技术边界与验收标准。

## 背景

当前仓库包含桌面应用 `apps/desktop` 和在线编辑器 `apps/web`，但没有独立产品官网。README 已形成清晰的产品方向：轻语是一款简单、实用、本地优先的 Markdown 编辑器，不以“第二大脑”、AI 或复杂知识管理为卖点。

产品官网需要把这一方向转化为面向新用户的完整叙事，并介绍编辑、个性化、同步、Web、桌面和移动端计划。官网不能把尚未发布的能力描述为已正式可用，也不能把营销页面耦合进在线编辑器的运行时。

## 目标

- 建立一个独立、可静态部署的轻语产品官网；
- 用“明窗净几，字字轻语。”建立明确的中文写作品牌气质；
- 完整介绍核心编辑能力、个性化、数据自主、WebDAV/S3 同步和多平台体验；
- 以桌面版下载为主要转化目标，以 Web 编辑器为次要入口；
- 中文为默认语言，同时支持手动切换英文；
- 在移动端正式发布前，用准确的“即将推出”状态介绍移动体验；
- 保持首屏轻量、响应式、可访问，并满足产品官网的基础 SEO 要求。

## 非目标

- 不修改 `apps/web` 在线编辑器或 `apps/desktop` 产品行为；
- 不在官网嵌入真实 Milkdown、CodeMirror 或 Tauri 运行时；
- 不提供账号、登录、评论、表单、邮件订阅、分析追踪或后台服务；
- 不创建尚不存在的 Android/iOS 下载入口；
- 不在本任务内决定生产域名、DNS、CDN 或具体托管供应商；
- 不重命名现有 `@markra/*` 包或兼容性敏感的技术标识。

## 已确认的产品决策

| 项目 | 决策 |
| --- | --- |
| 页面结构 | 单页产品官网 |
| 代码位置 | 新建独立 `apps/site` |
| 主视觉 | “纸墨刊物”方向 |
| 品牌标识 | 使用仓库现有轻语羽毛图标，不使用示意图标 |
| 官网强调色 | 轻语品牌暖金 |

## 单页信息架构

### 1. 顶部导航

导航包含产品、功能、同步、移动端、产品宣言、下载和语言切换。桌面使用轻量吸顶导航，窄屏折叠为可访问的菜单。所有站内入口滚动到同页锚点。

### 2. 首屏

首屏使用以下核心内容：

- 主标题：“明窗净几，字字轻语。”；
- 说明：轻语不尝试成为另一个“第二大脑”，而是提供一个能安心写字的地方；
- 主按钮：“免费下载桌面版”，滚动到下载区并优先展示当前系统；
- 次按钮：“打开 Web 编辑器”，进入现有在线编辑器；
- 右侧使用 HTML/CSS 构建轻语编辑器示例，显示 Newsprint 主题、文件树和中文写作内容。

首屏不加载真实编辑器代码，也不使用与实际产品不一致的概念界面。

### 3. 产品个性

用简短的刊物式段落说明：

- 不做第二大脑；
- 不依赖复杂的块与双链；
- 不加入 AI 负担；
- 内容始终是普通 Markdown 文件；
- 产品服务于记录和写作，而不是要求用户维护一套知识系统。

### 4. 核心功能

功能区覆盖：

- 所见即所得和源码模式；
- 普通文件与文件夹管理；
- 多标签、分栏、快速打开、工作区搜索和大纲；
- 链接、图片、HTML、KaTeX、Mermaid 和 GFM 表格；
- 自动保存、工作区恢复、字数统计和导出。

功能卡以真实产品能力为依据，不宣传仓库中不存在的能力。

### 5. 个性化体验

介绍主题、编辑器主题、字体、书写宽度、字号、行高和快捷键。该区强调“让工具适应写作者”，而不是展示复杂配置本身。

### 6. 同步与数据自主

同步区以“你的笔记，本该躺在你自己的存储桶里”为核心，说明：

- 文件默认保留在本地；
- 桌面版可为打开的项目文件夹配置 WebDAV 或 S3 兼容双向同步；
- 存储位置由用户控制；
- 同步用于跨设备延续，但不改变普通 Markdown 文件的可迁移性。

官网不得把同步描述成备份，也不得暗示 Web 运行时已经支持项目同步。

### 7. 移动端

移动区使用“案头挥毫，掌中轻语。”，展示 Compact 手机界面和桌面到手机的写作延续。状态必须明确标注“即将推出”。

仓库虽已有 Android/iOS 运行时与移动同步能力，但当前发布流程没有移动制品，因此本区不得出现 App Store、Google Play、APK 或 TestFlight 下载按钮。

### 8. 多平台与下载

正式可用的平台为 macOS、Windows、Linux 和 Web：

- 根据访问系统优先排列对应桌面卡片；
- 无法识别平台时展示全部平台；
- 下载入口使用项目现有正式发布地址；
- Web 卡片进入在线编辑器；
- Android/iOS 只作为“即将推出”展示，不混入可下载平台。

### 9. 产品宣言

页面后段完整展示：

> 我们并不需要另一个“第二大脑”，<br />
> 我们只需要一个能安心写字的地方。<br />
> 剥离复杂的块与双链，回归最纯粹的行云流水。<br />
> 数据归于你的 S3，灵感归于你的内心。<br />
> 在这里，只有你与文字的轻语。

### 10. 开源与页脚

页脚包含 GitHub、文档、隐私、更新日志、贡献指南和 AGPL-3.0 许可证入口。页尾再次提供桌面下载与 Web 编辑器入口，但不重复整段营销文案。

## 视觉系统

### 纸墨刊物方向

官网采用大面积暖白留白、细边框、刊物编号、小号英文标签和轻量网格。它应有中文独立刊物的编辑感，但不能牺牲软件产品信息的清晰度。

官网品牌强调色使用现有轻语暖金。此前视觉稿中的朱砂色不作为品牌色继续使用。

### 品牌标识

导航、首屏、favicon、下载卡和移动端预览统一使用仓库现有羽毛图标资产。不得以“羽”字、星形或通用钢笔图标代替品牌标识。

### 产品界面示例

首屏和功能区的界面示例由轻量 HTML/CSS 构建，使用 `packages/app/src/styles.css` 中现有 Newsprint 主题值：

- 纸张：`oklch(96% 0.018 88)`；
- 侧栏：`oklch(91% 0.022 86)`；
- 标题：`oklch(18% 0.012 75)`；
- 正文：`oklch(28% 0.012 75)`；
- 强调：`oklch(42% 0.12 24)`；
- 边框：`oklch(80% 0.025 78)`。

官网暖金和编辑器 Newsprint 强调色属于不同层级：前者表达轻语品牌，后者忠实呈现实际产品主题。

### 动效

只使用轻微淡入、滚动出现、导航状态和桌面到手机的短距离交接动效。禁止持续漂浮、复杂粒子、强视差和自动轮播。`prefers-reduced-motion` 下关闭非必要运动。

## 字体设计与加载

中文标题和正文使用用户指定的 [霞鹜文楷 GB Lite](https://github.com/lxgw/LxgwWenkaiGB-Lite)。英文标签、平台名称和技术性短文使用系统无衬线字体，避免为了少量英文扩大中文字体字符集。

生产站点只自托管 Regular 一个字重的 WOFF2 子集：

1. 以官方固定版本的 Regular TTF 为来源；
2. 收集最终中英文静态文案所需字形；
3. 生成 WOFF2 子集，不提交约 14 MB 的原始 TTF；
4. 使用不包含“霞鹜”或 `LXGW` 保留名称的内部衍生字体名；
5. 随站点保留 OFL-1.1 全文、来源版本和生成说明；
6. 首屏预加载该字体并设置 `font-display: swap`；
7. 未覆盖字符回退到 `Kaiti SC` 和系统中文字体。

设计预览中，当前页面字符生成的 WOFF2 为约 36 KB。最终文件大小以完整双语文案冻结后重新生成的结果为准，不预设无法验证的固定上限。

## 技术架构

### 独立应用

新建 `apps/site` pnpm workspace 包，使用 React、TypeScript、Vite 和 Tailwind CSS。官网不依赖 `@markra/editor`、Milkdown、CodeMirror、Tauri API 或桌面运行时。

根目录 `pnpm build` 通过现有递归脚本自动包含官网构建。官网提供独立的 `dev`、`build`、`test` 和 `typecheck:test` 脚本。

生产构建必须把默认中文页面预渲染进 `dist/index.html`，再由 React 在浏览器中接管语言切换和交互。静态主机不需要 Node 服务，禁用 JavaScript 时仍能看到核心产品介绍、宣言和普通链接。

### 组件边界

页面拆分为：

- `SiteHeader`；
- `Hero`；
- `ManifestoIntro`；
- `FeatureGrid`；
- `Personalization`；
- `SyncStory`；
- `MobilePreview`；
- `PlatformDownload`；
- `FullManifesto`；
- `SiteFooter`。

内容与视图分离。中英文内容位于一个类型安全的内容模型中，各语言必须覆盖同一字段集合，组件不得散落硬编码翻译。

### 语言状态

- 首次进入默认使用简体中文；
- 用户可在导航中手动切换英文；
- 选择保存在站点自己的 `localStorage` key 中；
- 存储不可用或值无效时回退中文；
- 不根据浏览器语言自动切换，也不改变 URL。

### 下载与外部入口

主 CTA 滚动到下载区。浏览器平台识别只用于排序或突出平台卡片，不隐藏其他正式平台。识别失败时展示全部桌面平台。

所有外部地址集中在单一配置模块中，包括 GitHub Releases、Web 编辑器、文档、隐私、更新日志和贡献指南，避免组件之间出现不一致链接。

### 无后端边界

官网没有账户、表单、订阅、追踪、远程内容接口或运行时 GitHub API 请求。构建输出是可部署到任意静态主机的 `dist/`。

## SEO 与分享

- HTML 提供中文默认的 `<title>`、description、canonical、favicon 和主题色；
- 提供 Open Graph 与 Twitter Card 元数据；
- 添加软件应用结构化数据，平台与下载状态必须和页面一致；
- 标题层级遵循单一 `h1` 与语义章节结构；
- 关键产品信息不能只存在于背景图或装饰性画面中；
- 默认中文核心内容预渲染进生产 HTML；
- 英文切换后的标题和可见文案由客户端更新，中文仍是默认可索引内容，英文不伪装成独立可索引 URL。

生产域名尚未指定。canonical、Open Graph URL 和 sitemap 的最终域名由部署配置注入，不在源代码中伪造占位域名。

## 响应式与无障碍

- 至少验证 390px、768px 和 1440px 视口；
- 手机首屏改为单列，编辑器示例置于文案之后；
- 导航折叠菜单支持键盘、焦点管理和 Escape 关闭；
- 所有按钮和链接有可见焦点状态；
- 图标按钮有可读标签，装饰图片使用空 `alt`；
- 正文、按钮、边框和主题示例满足合理对比度；
- 页面缩放与长英文内容不得造成横向溢出；
- 减少动画偏好下关闭非必要动效。

## 错误与降级

- 字体加载失败时使用系统楷体或中文系统字体，内容与布局仍可使用；
- 图片加载失败时保留文字说明，不隐藏功能内容；
- 平台识别失败时显示所有桌面下载项；
- `localStorage` 不可用时语言切换仅在当前页面生命周期内生效；
- 外部入口使用普通链接，不依赖脚本才能打开；
- 移动端没有可用下载时只显示“即将推出”，不生成空链接或占位按钮。

## 测试与验证

### 自动化测试

官网测试至少覆盖：

- 默认显示中文；
- 中英文切换并持久化；
- 无效或不可用语言存储回退中文；
- macOS、Windows、Linux 和未知平台的下载排序；
- Web 编辑器、下载、GitHub、文档和隐私链接来自统一配置；
- 移动端只显示“即将推出”，不存在 Android/iOS 下载链接；
- 移动导航打开、关闭、键盘与焦点行为；
- 各语言内容模型字段完整一致。

### 构建与体积

- `pnpm --filter @markra/site test`；
- `pnpm --filter @markra/site typecheck:test`；
- `pnpm --filter @markra/site build`；
- `pnpm build`；
- 检查 `dist/index.html` 在不执行 JavaScript 时仍包含主标题、核心产品说明和宣言；
- 验证官网构建产物不包含 Milkdown、CodeMirror 或 Tauri vendor chunk；
- 输出 JS、CSS、字体和图片体积清单；
- 确认首屏只预加载一份字体子集；
- 确认原始 TTF 未进入 `dist/`。

### 浏览器验收

至少在真实浏览器检查：

- 390px、768px、1440px 布局；
- 中文与英文内容切换；
- Newsprint 编辑器示例与品牌暖金的边界；
- 系统字体回退和字体加载完成后的布局稳定性；
- 键盘导航、焦点、减少动画偏好；
- 所有站内锚点与外部链接；
- macOS、Windows、Linux 和未知平台下载展示；
- 无控制台错误和横向溢出。

## 成功标准

- 官网作为独立 `apps/site` 构建，不增加编辑器包依赖；
- 单页完整覆盖产品个性、核心功能、个性化、同步、移动端、平台、宣言与下载；
- 默认中文、英文切换和语言持久化正常；
- 主 CTA 明确导向桌面下载，Web 编辑器为次入口；
- Android/iOS 明确标注“即将推出”，没有虚假下载入口；
- 使用轻语官方图标、Newsprint 实际主题配色和霞鹜文楷 GB Lite 子集；
- 字体和资源加载轻量，失败时可安全降级；
- 官网测试、官网构建和仓库根构建全部通过；
- 页面在手机、平板和桌面真实浏览器中可用且无明显可访问性问题。
