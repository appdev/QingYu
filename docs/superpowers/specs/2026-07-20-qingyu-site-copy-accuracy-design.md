# 轻语官网文案与功能叙事校正设计

## 状态

用户已要求继续优化官网中不准确的文案和功能介绍，并沿用此前“按推荐方案继续、无需询问”的授权。本设计在现有 Workbench 视觉系统内直接推进，不重新选择宏观布局、主题、页眉或页脚。

## 事实来源

官网文案以当前代码和运行时能力为准：

- `packages/app`：所见即所得与源码编辑、文件树、标签页、分栏、快速打开、工作区搜索、大纲、文件历史、字数统计、视图模式、主题与快捷键；
- `apps/desktop/src/runtime/desktop.ts`：桌面文件系统、导出、Pandoc、项目同步、MCP、系统字体、更新器等完整能力；
- `apps/web/src/runtime/index.ts`：浏览器文件句柄、编辑与 HTML/打印导出可用，项目同步、Pandoc 和 MCP 不可用；
- `packages/app/src/lib/project-config.ts`：WebDAV / S3 是按文件夹启用、默认关闭的项目级双向同步；
- `docs/qingyu-mcp.md`：桌面端 MCP 默认关闭，只能访问当前打开的项目，并由应用管理权限；
- `docs/testing/qingyu-mobile-native-results.md`：Android 模拟器和 iOS Simulator 已通过核心编辑烟测，但完整设备验收和公开发布仍未完成。

README 只作为事实交叉检查，不复制其中的营销段落，也不修改当前工作区内用户尚未提交的 README 改动。

## 内容原则

1. 标题、章节名和产品原则负责传达轻语安静、克制的诗意；紧邻正文负责说明产品、文件和平台边界，二者不互相替代。
2. “简单、实用、记录”是主定位；不再用“第二大脑”定义轻语，也不让功能清单磨平品牌气质。
3. 删除“剥离双链”等与当前双链补全和文档链接能力冲突的表述。
4. 不把同步描述成备份，也不暗示 Web 版或单独文件会启动项目同步。
5. 不把移动端描述成已公开可用；明确区分核心原生验证与正式发布。
6. MCP 作为默认关闭的桌面进阶能力出现，不包装成内置 AI 功能。
7. 中英文表达同一事实，但不做逐词硬译；CTA 使用明确动作。

## 页面叙事

### 首屏

- 中文标题：`明窗净几，字字轻语。`
- 英文标题：`A clear desk. An open file. Begin.`
- 正文用“打开一份 Markdown，文字便自然铺开”承接诗意，再明确所见即所得与源码模式共用普通文件、无需账号且没有云端锁定。
- 主 CTA 仍进入桌面下载区，次 CTA 仍打开 Web 编辑器。

### 产品定位

用“让记录回到纸页般自然”表达简单记录的选择。两段正文明确普通 Markdown、文件夹工作区、没有内置 AI、无需账号和文件可迁移性。

### 核心能力

按真实工作流介绍六组能力：

1. 文档视图、源码模式与左右分栏；
2. 文件树、多标签、快速打开、工作区搜索与大纲；
3. 链接、图片、HTML、GFM 表格、任务列表、KaTeX、Mermaid、提示块和代码高亮；
4. 桌面端自动保存、状态恢复、文件历史与字数统计；
5. 默认关闭、由应用授权、仅限当前项目的桌面端 MCP；
6. 普通 `assets/` 资源与桌面端 HTML、PDF、Pandoc 扩展格式导出。

### 个性化与视图

从泛化的主题宣传改为实际可用的完整、日常、专注、沉浸和自定义视图，并保留配色、字体、行高、书写宽度和可录制快捷键。

### 同步

同步图改为 `项目文件夹 ↔ WebDAV / S3`，不再使用会暗示所有设备和 Web 版都已参与的 `Your devices`。正文明确：

- 桌面端；
- 按文件夹单独启用；
- 默认关闭；
- WebDAV / S3 兼容双向同步；
- Web 版和单独文件不启动项目同步；
- 同步不是备份。

### 移动端

标题使用“案头之外，轻语正在走向掌中”，状态紧接着明确“原生验证中 · 尚未发布”：模拟器核心编辑、自动保存和恢复已通过，完整设备验收与公开发布仍在进行。继续不提供商店、APK 或 TestFlight 链接。

### 下载、原则与开源

- 下载标题使用“在桌面落笔，或从浏览器开始”，CTA 则保留明确动作；
- 桌面 CTA 使用“下载桌面版”，Web CTA 使用“打开编辑器”；
- 五行“产品宣言”改为五条兼顾诗意与可验证边界的产品原则；
- 开源区同时说明 AGPL-3.0 与普通文件格式的可迁移性。
