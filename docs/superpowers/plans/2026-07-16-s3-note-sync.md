# S3 Note Sync Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add S3-compatible note-folder synchronization as a fully equivalent peer to the existing WebDAV sync provider.

**Architecture:** Replace the WebDAV-specific native sync pipeline with a generic engine over a crate-private asynchronous `RemoteSyncBackend` trait. Keep WebDAV and S3 transport code in separate backends, share endpoint addressing and SigV4 signing with S3 image upload, and dispatch a discriminated provider request from the frontend runtime.

**Tech Stack:** React, TypeScript, Vitest, Tauri v2, Rust 2021, reqwest, quick-xml, HMAC-SHA256, pnpm.

## Global Constraints

- Use `pnpm` for JavaScript and frontend dependency workflows.
- Do not add the AWS SDK, `async-trait`, or another production dependency.
- Reuse the S3 connection configured under Storage; `SyncSettings.remotePath` is the note object prefix.
- S3 and WebDAV must use the same upload, download, deletion propagation, conflict, manual, save-after, and scheduled-sync behavior.
- Keep remote note sync desktop-only.
- Do not use the TypeScript `void` keyword or operator.
- Preserve existing user changes and do not stage `bg.png` or `logo.png`.
